# Student Social Network - Post Manager (JavaFX + JDBC)

## Опис проєкту
Цей проєкт — графічний застосунок (GUI) для управління дописами у локальній соціальній мережі для студентів. Додаток розроблено в рамках індивідуального проєкту з дисципліни "Об'єктно-орієнтоване програмування на мові Java
". 

Головна мета застосунку — надання зручного інтерфейсу для виконання базових CRUD-операцій (створення, читання, оновлення, видалення) над сутністю "Дописи" (Posts), а також реалізація пошуку за певними критеріями.

### Предметна область
Інформаційна система студентської соціальної мережі. Користувачі мають власні стіни, на яких можуть залишати дописи. Також підтримується механізм "лайків" та репостів (посилання на оригінальний допис). Застосунок фокусується на адмініструванні/управлінні таблицею дописів.

## Технологічний стек
* **Мова програмування:** Java 17+
* **Інтерфейс користувача:** JavaFX (з використанням FXML)
* **Робота з БД:** JDBC (виключно через `PreparedStatement` для запобігання SQL-ін'єкціям)
* **База даних:** PostgreSQL *
* **Збірка проєкту:** Maven
* **Архітектурний патерн:** MVC (Model-View-Controller)

## Схема Бази Даних (ER-діаграма)

Нижче наведена схема бази даних, що демонструє основні сутності та зв'язки між ними. Для роботи застосунку використовуються три таблиці:
* **users** — містить дані про користувачів системи (необхідна для забезпечення цілісності через зовнішні ключі).
* **posts** — основна таблиця, на якій фокусується робота застосунку (реалізація CRUD-операцій та пошуку).
* **reposts** — таблиця-зв'язка для фіксації поширень оригінальних дописів іншими користувачами.

## Функціонал застосунку
* **Create:** Додавання нового допису на стіну (із зазначенням ID автора та ID власника стіни).
* **Read / Search:** Перегляд загального списку дописів у вигляді таблиці. Реалізовано пошук за:
** ID або іменем автора.
** Фрагментом тексту (ключовим словом) у вмісті допису.
* **Update:** Редагування текстового вмісту існуючого допису.
* **Delete:** Видалення вибраного допису з бази даних.
* **UI/UX:** Виведення повідомлень про помилки (наприклад, невідомий ID користувача) та підтверджень дій через спливаючі вікна (Alerts).

## Технічне дослідження: PreparedStatement (Варіант №1)
У межах проєкту було досліджено механізм **PreparedStatement** як основний засіб взаємодії з СУБД PostgreSQL.
* **Безпека:** Використання параметризації (замість прямої конкатенації рядків) унеможливлює проведення SQL-ін’єкцій.
* **Ефективність:** Завдяки попередній компіляції SQL-запитів на стороні сервера, забезпечується вища продуктивність при повторному виконанні CRUD-операцій.
* **Практичне застосування:** Усі методи в `PostRepository.java` реалізовані через параметризовані запити, що відповідає кращим практикам розробки на Java.

## Архітектура застосунку
Проєкт побудовано за принципом **MVC**, що дозволило відокремити бізнес-логіку від інтерфейсу:
* **Model:** Клас `Post`, що описує структуру даних.
* **View:** `main_view.fxml` — опис графічного інтерфейсу.
* **Controller:** `PostController`, що керує взаємодією користувача з програмою.
* **DAO (Service):** `PostRepository`, що відповідає за низькорівневу роботу з JDBC.
